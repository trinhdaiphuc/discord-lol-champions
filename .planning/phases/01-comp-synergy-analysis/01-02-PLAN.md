<threat_model>
## Trust Boundaries

- Discord and HTTP callers can trigger repeated generation and therefore repeated persistence writes.
- Response formatting should expose only comp-analysis data, not raw internal persistence structure.

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| TM-04 | Denial of service | generate command persistence path | mitigate | Keep analysis synchronous but lightweight; avoid extra external calls and reuse in-memory champion data |
| TM-05 | Information disclosure | API response contract | mitigate | Return only summary/score fields, not internal DB IDs or storage-only signatures |
| TM-06 | Tampering | role-only generation integration | mitigate | Make partial-analysis behavior explicit in code and tests so misleading summaries are not silently shipped |

</threat_model>
