<threat_model>
## Trust Boundaries

- Discord commands and HTTP routes provide untrusted guild/context input to backend services.
- SQLite persistence is trusted only after service-level validation and normalized writes.

## STRIDE Register

| threat_id | category | component | disposition | mitigation_plan |
|-----------|----------|-----------|-------------|-----------------|
| TM-01 | Tampering | analysis-history writes | mitigate | Validate generation mode, guild ID, and serialized score payload shape before insert |
| TM-02 | Denial of service | repeated generate persistence path | mitigate | Keep writes bounded to one insert per generation and use indexed lookup fields only |
| TM-03 | Information disclosure | future history queries | mitigate | Persist only guild ID, champion IDs, numeric scores, summaries, and timestamps; do not store user PII |

</threat_model>
